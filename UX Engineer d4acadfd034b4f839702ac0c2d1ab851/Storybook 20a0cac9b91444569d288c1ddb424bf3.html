Introduction

Storybook runs alongside your app in development mode. It will help you build UI component and complex layout isolated from business logic and context of your app.

Please refer to the official docs https://github.com/storybookjs/react-native

  • Installation Step
    • Run this command to install storybook
    npx -p @storybook/cli sb init --type react_native
    • Accept it if it is asking to install storybook server, if not, run this command:
    yarn add @storybook/react-native-server
    • Add this code on your entrypoint:
    export {default} from "./storybook";
    • Run storybook server
    yarn start-storybook

or add script to your package.json

"storybook": "start-storybook"
    • Storybook configuration
    import {AppRegistry} from 'react-native';
import {getStorybookUI, configure} from '@storybook/react-native';

import './rn-addons';

configure(() => {
  require('./stories');
}, module);

const StorybookUIRoot = getStorybookUI({
  asyncStorage: null,
  port: 7007,
});

AppRegistry.registerComponent('%APP_NAME%', () => StorybookUIRoot);

export default StorybookUIRoot;
    • There is configuration to choose either you run Storybook or the apps.
    const root = IS_STORYBOOK ? Storybook : projects;
export default root;

Creating Component

Component could be made either a class component or functional component.

  • create a components folder in /src, use general name for general component and add feature name if the component is for specific feature.
  • create story folder inside /storybook/stories, try to group it to make it more organize. You will call the component from here to run in storybook.
  • last step, add the new stories to storybook list in storybook/stories/index.js
    import React from 'react';

import {storiesOf} from '@storybook/react-native';
import {linkTo} from '@storybook/addon-links';
import {
  withKnobs,
} from '@storybook/addon-knobs';

import CenterView from './CenterView';
import Welcome from './Welcome';

//new component
import ButtonStory from './Button';

Connecting Component to Zeplin

  • Open Zeplin
  • Select Styleguides and right Project, MySF_Component for component use adobe XD, MySF_Figma_Library for component use Figma. We are migrating to Figma and soon will leave adobe XD.
  • Select the component, get the component ID or component Name
  • Open the file .zeplin/components.json, append the new component to the last item following the previous format.
  • Run zeplin connect on your project, If success, you will get the following screenshoot. If not, read the error log and try to fix it.

Development Lifecycle

  • Create a branch off of master

    Name the branch with your first name pre-pended:

    story/cool-feature

    Commit your work frequently and use descriptive comment. When you are finish, rebase your work to master branch and squash the commit message into one commit message per ticket (ussually you will have one ticket per branch)

    Push your branch onto remote repository at the end of the days if you have not finished yet.

  • Testing
    • Test your component on android in screen size 720x1280 and 1080x2280, iPhone6 and iPhone 11, iPad.
    • Make screenshoot for each screen and try to use Larger Font from your device, see if the layout is breaking or not.
    • Try to use long text for for each label and make sure the layout don't break.
  • Create a Merge Request on Gitlab

    Include the screenshoot or video of your component.

  • Submit for review

    Add additional approver if you needed.

Scalable Font Size

Function to create scalable font size in MySF, textAdjust accept size that will be scaled and will calculate the new size based on device screen width. This function also applicable to other component with fixed size to make it responsive.

import {SCREEN_WIDTH} from 'helpers/screenSize';

export function textAdjust(size: number) {
  const MAX_SCALE_FACTOR = 1.5;
  const BASE_SCREEN_SIZE = 360;
  const scaleFactor = SCREEN_WIDTH / BASE_SCREEN_SIZE;
  return Math.min(scaleFactor, MAX_SCALE_FACTOR) * size;
}

SCREEN_WIDTH is const that will return device screen width.

MAX_SCALE_FACTOR is maximum scale factor if the screen is too big (iPad and Tablet)

BASE_SCREEN_SIZE is screen width that designer use on their phone screen design.

How to Use

<Text style={{fontSize: textAdjust(16)}} />

<View style={{width: textAdjust(320), height: textAdjust(56)}} />

Guided Path use modal from react-native-modal to show the highlighted component, in order to render it properly, the component need time and make sure the highlighted component is rendered first. If the component is changing position while it rendering, the Guided path will also change the highlighted position when it is visible.

Issues

Scrollable Tab View

TabView component is a library from react-native-tab-view v2.15.2 and there are the swipe gesture function to navigate between tab.

Horizontal scrollable component will accidentally switch tab when you are horizontally swipe inside the component.

Solution

Use ScrollView or FlatList from react-native-gesture-handler inside TabView

Guided Path

Translucent StatusBar

Translucent StatusBar on Android will make incorrect position of the Guided Path

Solution

Set props isStatusBarTranslucent on <ToolTipV2 /> to true if the StatusBar is translucent.

Animation

Loop animation will prevent Guided Path visible until the animation finish.

GuidedPath use InteractionManager to wait initial interaction from others component before it's rendering completely and by default, React Native, set Animated Component as interaction.

Solution

set isInteraction to false if you have component with Animated function and no need to wait the animation until finished before showing the GuidedPath.

set isInteraction on loop animation

Timer is Reset After finish.

If you use component <CountDownSF /> or <CardSmartTimer /> inside component, probably there will be issue if you do not carefully check the component lifecycle. Arrow function will create new function every time the component re-render.

Solution

Do not put <CountDownSF /> or <CardSmartTimer /> inside arrow function if the component re-render after the timer is finished.

Auto Height BottomSheet

There are two components to create BottomSheet, <SmartBottomSheet /> and <ScrollableModal />.

props autoHeight in SmartBottomSheet doesn't always work, it will give transparent empty space at the bottom on Android.

Solution

Use ScrollableModal if props autoHeight doesn't always work.

Running Animation

React Native can run animation using native component by set isNativeDriver to true, but it only possible if the animation do not change the layout of the component, like rotation, opacity and translation. Updating the height or width of the view using animation probably will impact the app performance because the animation cannot use native component and will consume JS Thread.

Solution

Try to avoid animation that updating the layout of the view so you can set useNativeDriver to true.

Margin inside SVG Animation

react-native-lottie will not remove empty spacer or margin automatically on SVG Animation

Solution

Make the SVG animation with no margin.
